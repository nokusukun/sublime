# Security and correctness slop

The dangerous category. Every pattern here is code that compiles, passes a casual review, and fails under adversarial input, scale, or load. Empirical studies put the LLM-generated vulnerability rate at ~40% across 1,689 Copilot programs (Pearce et al., *Asleep at the Keyboard*), and the Stanford AI-assistance study found developers produced less secure code while believing they produced more. Treat every entry here as a hard ban with narrow, documented exceptions.

### Fabricated API call

**Tags:** `AI-slop` · `Lint` · `Universal`

**Pattern.** A call to a function, method, or module that does not exist in the version of the library actually installed.

**Forbidden example.**
```ts
import crypto from "node:crypto";

export function seal(data: string, key: Buffer) {
  return crypto.magicallyEncrypt(data, key);
}
```

**Why it hurts.** The curl team received an AI-generated vulnerability report that referenced "a function that does not exist in cURL at all." Models confidently invent method names on real packages as often as they invent packages. The code reads plausibly, the import resolves, and the call fails only at runtime — often in production, on the one path tests didn't cover.

**Rewrite.**
```ts
import { createCipheriv, randomBytes } from "node:crypto";

export function seal(data: string, key: Buffer) {
  const iv = randomBytes(12);
  const cipher = createCipheriv("aes-256-gcm", key, iv);
  const enc = Buffer.concat([cipher.update(data, "utf8"), cipher.final()]);
  return { iv, ciphertext: enc, tag: cipher.getAuthTag() };
}
```

### Slopsquatting / hallucinated import

**Tags:** `AI-slop` · `Lint` · `Universal`

**Pattern.** An `import` statement naming a package that does not exist on the registry.

**Forbidden example.**
```ts
import { fastEncrypt } from "fast-crypto-utils";
import mongoose from "express-mongoose";
```

**Why it hurts.** Spracklen et al. (USENIX '25) found 19.7% of 576,000 AI-generated samples referenced non-existent packages; 43% of those names repeat across runs. Seth Larson coined "slopsquatting" for the attack class where adversaries register the hallucinated names. Real incidents on crates.io (`faster_log`), npm (`unused-imports`), and RubyGems (`arangodb`) have already shipped malware this way. An unverified import is a supply-chain vulnerability.

**Rewrite.**
```ts
import { createCipheriv, randomBytes } from "node:crypto";
```

### SQL-string-concat

**Tags:** `AI-slop` · `Lint` · `Universal`

**Pattern.** Building a SQL query by concatenating or interpolating untrusted input into the query string.

**Forbidden example.**
```ts
export async function findUser(id: string) {
  return db.query("SELECT * FROM users WHERE id = " + id);
}
```

**Why it hurts.** This is classic SQL injection. Pearce et al. found ~40% of 1,689 Copilot-generated programs vulnerable; SQLi and command injection were among the most prevalent classes. Every ORM and query builder in mainstream languages has a safe parameterized path; using string concat reverts to 1998. An attacker passes `1 OR 1=1 --` and reads the table.

**Rewrite.**
```ts
export async function findUser(id: string) {
  return db.query("SELECT * FROM users WHERE id = $1", [id]);
}
```

**See in `/sublime`:** [`../sublime/SKILL.md#interfaces--boundaries`](../sublime/SKILL.md#interfaces--boundaries), [`../sublime/references/data-modeling.md`](../sublime/references/data-modeling.md).

### Math.random for security tokens

**Tags:** `AI-slop` · `Lint` · `Universal`

**Pattern.** A non-cryptographic RNG used to generate session IDs, password-reset tokens, CSRF tokens, or API keys.

**Forbidden example.**
```ts
export function makeResetToken() {
  return Math.random().toString(36).slice(2);
}
```

**Why it hurts.** `Math.random()` is a PRNG with ~52 bits of exposed state in V8 and is predictable from a handful of consecutive outputs. Anything generated by it and used for authentication, authorization, or session integrity is forgeable. This is BAN 10 in the Sublime foundation — there is no legitimate use of `Math.random` for security-meaningful values.

**Rewrite.**
```ts
import { randomBytes } from "node:crypto";

export function makeResetToken() {
  return randomBytes(32).toString("base64url");
}
```

**See in `/sublime`:** [`../sublime/SKILL.md#hard-bans`](../sublime/SKILL.md#hard-bans).

### Hardcoded secret in source

**Tags:** `AI-slop` · `Lint` · `Universal`

**Pattern.** An API key, connection string, or credential inlined as a literal in application code.

**Forbidden example.**
```python
STRIPE_KEY = "sk_live_51H8xP9J..."
DATABASE_URL = "postgres://admin:hunter2@prod-db.internal:5432/app"

stripe.api_key = STRIPE_KEY
```

**Why it hurts.** Variant Systems' audit found hardcoded secrets "across every Claude Code project" — the model saw the secret in prompt context and baked it in. Once in git history, the secret is compromised forever, even after rotation and `git filter-branch`. Every major secret-scanner (gitleaks, truffleHog, GitHub's own push-protection) catches these, but the damage is done at commit time.

**Rewrite.**
```python
import os

stripe.api_key = os.environ["STRIPE_KEY"]
DATABASE_URL = os.environ["DATABASE_URL"]
```

### Bypassing framework CSRF/CORS

**Tags:** `AI-slop` · `Review` · `Universal`

**Pattern.** Disabling or wildcarding a framework's built-in cross-origin or CSRF protection to "make it just work."

**Forbidden example.**
```ts
import cors from "cors";

app.use(cors({ origin: "*", credentials: true }));
app.disable("csrf");
```

**Why it hurts.** `origin: "*"` combined with `credentials: true` is either silently ignored by the browser or, when honored, hands your authenticated session to every website on the internet. Frameworks ship CSRF and CORS defaults specifically because getting them wrong is catastrophic. The "works on localhost" prompt that produced this code will silently ship to production.

**Rewrite.**
```ts
import cors from "cors";

app.use(cors({
  origin: ["https://app.example.com"],
  credentials: true,
}));
```

### N+1 query

**Tags:** `AI-slop` · `Review` · `Universal`

**Pattern.** A loop that issues one database query per item instead of a single bulk query.

**Forbidden example.**
```python
orders = Order.objects.filter(user=user)
for order in orders:
    order.product = Product.objects.get(id=order.product_id)
    order.shipping = Shipping.objects.get(order_id=order.id)
```

**Why it hurts.** Sohail: *"AI-generated code is almost always written for a single request, a single user, a small dataset."* A page of 50 orders becomes 101 queries. Latency scales linearly, the database connection pool exhausts, and p99 falls off a cliff under load. Works on `pytest`, breaks on Black Friday.

**Rewrite.**
```python
orders = (
    Order.objects
    .filter(user=user)
    .select_related("product")
    .prefetch_related("shipping")
)
```

### No pagination / unbounded fetch

**Tags:** `AI-slop` · `Review` · `Universal`

**Pattern.** A query with no `LIMIT`, no page size, and no cursor — fetching "all rows" unconditionally.

**Forbidden example.**
```ts
export async function listUsers() {
  return db.users.find({});
}
```

**Why it hurts.** On day one, ten rows. On day 400, ten million. The function that ran in 2ms now OOMs the process and times out every caller. The fix is trivial at design time and expensive after launch, because every caller already relies on "returns everything." Unbounded reads are a latent incident.

**Rewrite.**
```ts
export async function listUsers({ cursor, limit = 100 }: ListUsersOptions) {
  const rows = await db.users
    .find(cursor ? { _id: { $gt: cursor } } : {})
    .limit(limit + 1)
    .toArray();
  const hasMore = rows.length > limit;
  return { users: rows.slice(0, limit), nextCursor: hasMore ? rows[limit - 1]._id : null };
}
```

### Unbounded goroutine / unbounded worker

**Tags:** `AI-slop` · `Review` · `Lang:go`

**Pattern.** Spawning one goroutine (or worker, task, promise) per item in a collection with no semaphore, bound, or cancellation.

**Forbidden example.**
```go
func processAll(items []Item) {
    for _, item := range items {
        go process(item)
    }
}
```

**Why it hurts.** With ten items, fine. With ten million, you have ten million goroutines fighting for scheduler time, ten million open DB connections, and a panic when the connection pool exhausts. There is no backpressure, no cancellation on failure, no way to wait for completion, and no ordering guarantee. A bounded worker pool is barely more code.

**Rewrite.**
```go
func processAll(ctx context.Context, items []Item) error {
    g, ctx := errgroup.WithContext(ctx)
    g.SetLimit(16)
    for _, item := range items {
        item := item
        g.Go(func() error { return process(ctx, item) })
    }
    return g.Wait()
}
```

**See in `/sublime`:** [`../sublime/SKILL.md#control-flow--structure`](../sublime/SKILL.md#control-flow--structure), [`../sublime/references/control-flow.md`](../sublime/references/control-flow.md).

### Stanford "insecure-but-confident" effect

**Tags:** `AI-slop` · `Review` · `Universal`

**Pattern.** Code that looks clean, passes tests, and the author is confident in — but has a security bug the author did not consider because the AI did not surface it.

**Forbidden example.**
```ts
export async function downloadFile(req, res) {
  const name = req.query.name;
  const path = `./uploads/${name}`;
  res.sendFile(path);
}
```

**Why it hurts.** Perry et al. (Stanford) showed developers with AI assistance "produced less secure code while believing it was more secure." The confidence gap is itself the pattern — the code reads fine, the AI did not warn, the author did not audit. Here, `name = "../../../etc/passwd"` is a path traversal. The AI produced plausible code; the human shipped it because nothing in the loop forced them to ask "what if the input is adversarial?"

**Rewrite.**
```ts
import path from "node:path";

export async function downloadFile(req, res) {
  const name = path.basename(String(req.query.name));
  const full = path.resolve("./uploads", name);
  if (!full.startsWith(path.resolve("./uploads") + path.sep)) {
    return res.status(400).end();
  }
  res.sendFile(full);
}
```

### Assumption propagation (Karpathy)

**Tags:** `AI-slop` · `Review` · `Universal`

**Pattern.** Baking assumed values — hostnames, timeouts, retry counts, environment names — into literals instead of surfacing them as configuration or asking the user.

**Forbidden example.**
```python
def connect():
    return psycopg.connect(
        "postgres://app@db.internal:5432/production",
        connect_timeout=5,
    )

def fetch_with_retry(url):
    for _ in range(3):
        try:
            return requests.get(url, timeout=10).json()
        except Exception:
            continue
```

**Why it hurts.** Karpathy: *"The models make wrong assumptions on your behalf and run with them without checking. They don't manage confusion, don't seek clarifications, don't surface inconsistencies."* The hostname, the port, the retry count, the timeout, the database name — each is a guess the model made silently. Shipping means shipping those guesses. Surface them to the caller or fail loudly when they are missing.

**Rewrite.**
```python
def connect(dsn: str, *, timeout: float):
    return psycopg.connect(dsn, connect_timeout=timeout)

def fetch_with_retry(url: str, *, retries: int, timeout: float):
    for attempt in range(retries):
        try:
            return requests.get(url, timeout=timeout).json()
        except RequestException:
            if attempt == retries - 1:
                raise
```
